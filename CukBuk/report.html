  • A for HTML

    • We've done lots of work in HTML. We build almost everything by ourselves. Though, we searched some frameworks and tutorials to start with and for special design purpose. Literaturally, we build the webpages element by element, line by line.

  • A for CSS

    • We used Bootstrap css framework at first. Then, we understand most of css styles and wrote our own css file. We changed the padding-top, the front color and size, background, and so on. We defined our own .class element to reuse the style file. Moreover, we also apply a set of linear gradient background color to adapt to different browser.

  • B for JS

    • For javascript, we applied plenty of frameworks such as Bootstrap and JQuery. Also, we created our own javascript file as well for data transfer to backend and basic css changes. We tried to use Ajax for data packaging and transfer to backend and receiving from responses.

  • X for PNG
  • X for SVG
  • C for Server

    • What we are trying to do here is to build a server using express. Most of the tutorials online use express package to generate .jsp layouts. We build the frontend first with .html, thus the server will be used to receive requests such as register, login, search from database and display on the relative webpage. For now, our server is basic, which means we only finish the part to set up the server.

  • A for Database
  • C for Dynamic pages

    • First of all, we applied a carousel slide to show dynamic recommendations from database in the homepage. Also, we tried to get dynamic pages from database. For example, a search page to represent different recipes from database everytime we search for the keywords. Or, a webpage for creating recipes, store them to the database, and show them in a dynamic recipe page.

  • C for Depth (out of 40)

    • Our ultimate goal is to build a recipe website for the purpose of creating and sharing homemade recipes and shop for the ingradients. The website is mainly designed for desktops. What we are tring to do is to dynamically create recipe templates, store them to database. Thus, people can share and search for the recipes. Apart from that, we also want to build a shopping system for people to shop for the ingradients used in recipes. We've done lots of work in understanding html, css, js, png, database, and dynamic pages. However, with over 50 hours work on frontend and database, we only finished a well-designed website and a database. What we don't have time to go deep during examination period is the server. We realised that there are lots of tutorials online, however, none of them satisfies our purpose smoothly. For example, when we want to build the server to request from the modal inside the webpage, it takes a lot of effort and time to find out how to do that.
